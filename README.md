> Foundations of a man take time.

I build production-minded experiments that start as weird but useful product questions, then become shipped apps, agent workflows, payment rails, games, or native software. The throughline is simple: give software more agency, make money movement safer, and keep the experience human enough that people actually want to use it.

## Operating Style

- Ship the product path, not just the demo path.
- Keep agent money flows auditable, bounded, and explicit.
- Build for visible behavior first: users feel the truth of software before they read the architecture.
- Stay close to African consumer contexts instead of sanding everything down into generic global SaaS.
